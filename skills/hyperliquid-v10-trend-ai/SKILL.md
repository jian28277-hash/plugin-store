# Hyperliquid V10.0 趋势跟踪AI策略

## 策略简介

本策略是基于 **Hyperliquid Plugin** 构建的AI趋势跟踪交易系统，通过多维度评分模型自动筛选最优交易标的，在Hyperliquid DEX上自动执行交易。

**核心优势：**
- 🧠 AI四维度评分：趋势+动量+突破+波动率
- 🎯 高确信信号：评分≥70分才开仓
- 🛡️ 智能风控：2%止损+6%止盈+移动保本
- 📊 全市场扫描：BTC/ETH/SOL等主流币
- ⚡ 自动执行：识别信号→Hyperliquid下单→自动管理

---

## 第一步：数据采集（每30分钟）

通过 **Hyperliquid Plugin** 获取数据：

**1. K线数据**
```bash
hyperliquid-plugin 获取K线 --币种 BTC --周期 15m --数量 50
```

**2. 技术指标计算**
- EMA12/EMA26：判断短期和中期趋势
- RSI(14)：判断超买超卖
- ATR(14)：衡量波动率大小
- 成交量：确认突破真假

**3. 账户状态**
```bash
hyperliquid-plugin 查询余额
hyperliquid-plugin 查询持仓
```

## 第二步：AI四维度评分系统

### 维度1：趋势强度（40分）

| 条件 | 得分 | 说明 |
|------|------|------|
| 价格>短期均线>长期均线 | +40 | 多头排列，强势上涨 |
| 价格>短期均线 | +20 | 一般上涨 |
| 价格<短期均线<长期均线 | -40 | 空头排列，强势下跌 |
| 价格<短期均线 | -20 | 一般下跌 |

### 维度2：动量确认（25分）

| RSI区间 | 得分 | 说明 |
|---------|------|------|
| < 30 | +25 | 深度超卖，反弹概率高 |
| 30-40 | +15 | 超卖区域，可能反弹 |
| 40-60 | 0 | 震荡区间，无方向 |
| 60-70 | +15 | 强势延续 |
| > 70 | -10 | 超买区域，谨慎追多 |

### 维度3：突破信号（30分）

| 条件 | 得分 | 说明 |
|------|------|------|
| 突破6根K线高点+0.3%且放量 | +30 | 真突破，做多 |
| 跌破6根K线低点-0.3%且放量 | -30 | 真跌破，做空 |
| 突破但无量 | +10 | 假突破嫌疑 |
| 无突破 | 0 | 观望 |

### 维度4：波动率过滤（5分）

| 波动率 | 得分 | 说明 |
|--------|------|------|
| 0.5% - 3% | +5 | 正常波动，适合交易 |
| > 3% | -5 | 波动太大，减小仓位 |
| < 0.3% | -10 | 没波动，不交易 |

## 第三步：AI综合决策

### 推理1：计算总分

```
总分 = 趋势分 + 动量分 + 突破分 + 波动率分
```

### 推理2：方向判断

| 总分 | 1小时趋势 | 15分钟方向 | AI决策 |
|------|----------|-----------|--------|
| ≥70 | 上涨 | 上涨 | **做多** |
| ≥70 | 下跌 | 下跌 | **做空** |
| ≥70 | 上涨 | 下跌 | **不做**（逆势） |
| ≥70 | 下跌 | 上涨 | **不做**（逆势） |
| <70 | 任意 | 任意 | **观望** |

### 推理3：仓位计算

```
基础仓位 = 账户余额 × 20%
特殊情况调整：
- 波动率>3% → 仓位减半（10%）
- RSI<25或>75 → 仓位减半（10%）
- 连续亏损2次 → 仓位减半（10%）
```

### 推理4：开仓确认清单

- [ ] 总分≥70分
- [ ] 1小时趋势和15分钟方向一致
- [ ] 成交量确认突破（放量1.2倍）
- [ ] 当前持仓<2个币
- [ ] 今日交易<3次
- [ ] 账户余额>50USDC

## 第四步：通过Hyperliquid执行交易

### 开仓流程

1. **设置杠杆**
   ```bash
   hyperliquid-plugin 设置杠杆 --币种 BTC --杠杆 5
   ```

2. **市价开仓**
   ```bash
   hyperliquid-plugin 下单 --币种 BTC --方向 做多 --数量 0.05 --类型 市价 --策略ID hyperliquid-v10-trend-ai
   ```
   ⚠️ **必须加 --策略ID，否则不计入比赛成绩**

3. **记录状态**
   - 入场价格
   - 止损价格（入场价-2%）
   - 止盈价格（入场价+6%）

### 持仓管理（每5分钟检查）

**1. 止损检查**
```
如果 当前价格 ≤ 止损价:
    执行：hyperliquid-plugin 平仓 --币种 BTC
    记录亏损
    今日交易次数+1
```

**2. 移保本检查**
```
如果 盈利 ≥ 3% 且 未移保本:
    止损价 = 入场价（保证不亏）
    标记已移保本
```

**3. 移动止盈检查**
```
如果 盈利 ≥ 5%:
    记录最高盈利点
    如果 从最高点回撤 3%:
        执行：hyperliquid-plugin 平仓 --币种 BTC
```

**4. 固定止盈检查**
```
如果 盈利 ≥ 6%:
    执行：hyperliquid-plugin 平仓 --币种 BTC
    记录盈利
    今日交易次数+1
```

## 第五步：风控体系

### 1. 单笔风控

| 项目 | 设置 | 说明 |
|------|------|------|
| 止损 | 2% | 错单快速止损 |
| 止盈 | 6% | 盈亏比3:1 |
| 最大亏损 | 账户3% | 单笔风险严格控制 |

### 2. 日度风控

| 项目 | 设置 | 说明 |
|------|------|------|
| 最大交易次数 | 3次 | 防止过度交易 |
| 最大持仓数 | 2个 | 集中管理 |
| 日亏损上限 | 15% | 当天停止 |

### 3. 余额保护

```
如果 账户余额 < 50USDC:
    停止所有交易
    保留最后本金
```

### 4. 自适应调整

| 情况 | 调整 |
|------|------|
| 波动率>3% | 仓位减半 |
| RSI极端(<25或>75) | 仓位减半 |
| 连续亏损2次 | 仓位减半 |

## 使用示例

### 场景1：比特币做多

**AI分析：**
- 趋势分：+40（多头排列）
- 动量分：+15（RSI从超卖反弹）
- 突破分：+30（放量突破）
- 波动率：+5（正常波动）
- **总分：90分** ✅

**AI决策：**
1. 总分90≥70，满足条件
2. 1小时趋势上涨，15分钟也上涨
3. 成交量放大1.5倍，确认突破
4. 当前持仓0个，今日交易0次

**执行：**
```bash
# 设置杠杆
hyperliquid-plugin 设置杠杆 --币种 BTC --杠杆 5

# 开仓做多
hyperliquid-plugin 下单 --币种 BTC --方向 做多 --数量 0.05 --类型 市价 --策略ID hyperliquid-v10-trend-ai

# 自动设置：
# 止损价 = 入场价 × 0.98
# 止盈价 = 入场价 × 1.06
```

**持仓管理：**
- 价格上涨3% → 止损移到入场价（保本）
- 价格上涨5% → 启动移动止盈
- 价格上涨6% → 触发止盈，平仓获利
- 价格下跌2% → 触发止损，平仓止损

### 场景2：以太坊震荡行情

**AI分析：**
- 趋势分：+20
- 动量分：0
- 突破分：0
- 波动率：-10
- **总分：10分** ❌

**AI决策：**
- 总分10分<70分
- **结论：观望，不操作**

## 错误处理

| 错误 | 原因 | 解决办法 |
|------|------|---------|
| "评分不足" | 市场没趋势 | 等30分钟再分析 |
| "逆势交易" | 大小周期冲突 | 跳过，等方向一致 |
| "已达持仓上限" | 已有2个持仓 | 平仓一个或等待 |
| "今日交易已满" | 已做3笔 | 明天再交易 |
| "余额不足" | 资金<50USDC | 充值或减少仓位 |
| "API超时" | 网络问题 | 10秒后重试 |

## 安全提示

- **风险等级**：高级 - 本策略自动执行真实交易
- **默认模式**：模拟交易（--dry-run），实盘需关闭
- **建议杠杆**：5倍（可调，不建议超过10倍）
- **止损强制**：2%止损不能关闭
- **免责声明**：过往业绩不代表未来，加密货币交易可能损失全部本金
