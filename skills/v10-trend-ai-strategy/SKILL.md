# V10.0 全市场趋势跟踪AI策略

## 策略简介

本策略是一个AI驱动的趋势跟踪交易系统，通过多维度评分模型自动筛选全市场最优交易标的，实现智能开仓、动态止盈止损、严格风控的完整闭环。

**核心优势：**
- 🧠 AI四维度评分：趋势+动量+突破+波动率
- 🎯 高确信信号：评分≥70分才开仓
- 🛡️ 智能风控：2%止损+6%止盈+移动保本
- 📊 全市场扫描：8个主流币，每30分钟分析
- ⚡ 自动执行：识别信号→自动下单→自动管理

---

## 第一步：数据采集（每30分钟）

策略自动获取以下数据：

**1. K线数据**
- 15分钟K线（找入场信号）
- 1小时K线（定大趋势方向）
- 最近48根K线计算指标

**2. 技术指标**
- EMA12/EMA26：判断短期和中期趋势
- RSI(14)：判断超买超卖
- ATR(14)：衡量波动率大小
- 成交量：确认突破真假

**3. 市场状态**
- 当前持仓数量
- 当日交易次数
- 账户余额

## 第二步：AI四维度评分系统

策略对每币种进行100分制评分：

### 维度1：趋势强度（40分）

| 条件 | 得分 | 说明 |
|------|------|------|
| 价格>短期均线>长期均线 | +40 | 多头排列，强势上涨 |
| 价格>短期均线 | +20 | 一般上涨 |
| 价格<短期均线<长期均线 | -40 | 空头排列，强势下跌 |
| 价格<短期均线 | -20 | 一般下跌 |

**AI判断逻辑：**
- 均线多头排列 = 大势向上 → 优先考虑做多
- 均线空头排列 = 大势向下 → 优先考虑做空
- 均线纠缠 = 震荡 → 观望不操作

### 维度2：动量确认（25分）

| RSI区间 | 得分 | 说明 |
|---------|------|------|
| < 30 | +25 | 深度超卖，反弹概率高 |
| 30-40 | +15 | 超卖区域，可能反弹 |
| 40-60 | 0 | 震荡区间，无方向 |
| 60-70 | +15 | 强势延续 |
| > 70 | -10 | 超买区域，谨慎追多 |

**AI判断逻辑：**
- RSI<30 = 跌过头了 → 准备抄底做多
- RSI>70 = 涨过头了 → 准备做空或观望
- RSI 40-60 = 没方向 → 等突破

### 维度3：突破信号（30分）

| 条件 | 得分 | 说明 |
|------|------|------|
| 突破6根K线高点+0.3%且放量 | +30 | 真突破，做多 |
| 跌破6根K线低点-0.3%且放量 | -30 | 真跌破，做空 |
| 突破但无量 | +10 | 假突破嫌疑，降低分数 |
| 无突破 | 0 | 观望 |

**AI判断逻辑：**
- 放量突破 = 资金进场 → 跟随趋势
- 无量突破 = 诱多/诱空 → 不参与或降低仓位
- 没有突破 = 震荡 → 等待

### 维度4：波动率过滤（5分）

| 波动率 | 得分 | 说明 |
|--------|------|------|
| 0.5% - 3% | +5 | 正常波动，适合交易 |
| > 3% | -5 | 波动太大，减小仓位 |
| < 0.3% | -10 | 没波动，不交易 |

**AI判断逻辑：**
- 波动适中 = 有空间赚钱 → 正常操作
- 波动太大 = 风险过高 → 降低仓位或跳过
- 波动太小 = 没机会 → 不浪费资金

## 第三步：AI综合决策（核心）

### 推理1：计算总分

```
总分 = 趋势分 + 动量分 + 突破分 + 波动率分
```

**示例：**
- BTC当前：趋势+40，动量+15，突破+30，波动率+5 = **90分** ✅
- ETH当前：趋势+20，动量+0，突破+10，波动率-5 = **25分** ❌

### 推理2：方向判断

| 总分 | 1小时趋势 | 15分钟方向 | AI决策 |
|------|----------|-----------|--------|
| ≥70 | 上涨 | 上涨 | **做多** |
| ≥70 | 下跌 | 下跌 | **做空** |
| ≥70 | 上涨 | 下跌 | **不做**（逆势） |
| ≥70 | 下跌 | 上涨 | **不做**（逆势） |
| <70 | 任意 | 任意 | **观望** |

**关键原则：不做逆势单！**

### 推理3：仓位计算

```
基础仓位 = 账户余额 × 20%

特殊情况调整：
- 波动率>3% → 仓位减半（10%）
- RSI<25或>75 → 仓位减半（10%）
- 连续亏损2次 → 仓位减半（10%）
```

### 推理4：开仓确认清单

AI必须全部检查通过才开仓：

- [ ] 总分≥70分
- [ ] 1小时趋势和15分钟方向一致
- [ ] 成交量确认突破（放量1.2倍）
- [ ] 当前持仓<2个币
- [ ] 今日交易<3次
- [ ] 账户余额>50USDT

**任一条件不满足 → 跳过本次，继续等待**

## 第四步：自动执行交易

### 开仓流程

1. **设置杠杆**
   ```bash
   okx swap set_leverage --instId BTC-USDT-SWAP --lever 5 --mgnMode cross
   ```

2. **市价开仓**
   ```bash
   okx swap place_order --instId BTC-USDT-SWAP --side buy --ordType market --sz 0.01 --tdMode cross --tag agentTradeKit
   ```
   ⚠️ **必须加 --tag agentTradeKit，否则不计入比赛成绩**

3. **记录状态**
   - 入场价格
   - 止损价格（入场价-2%）
   - 止盈价格（入场价+6%）
   - 移保本触发点（入场价+3%）

### 持仓管理（每5分钟检查）

**1. 止损检查**
```
如果 当前价格 ≤ 止损价:
    立即平仓
    记录亏损
    今日交易次数+1
```

**2. 移保本检查**
```
如果 盈利 ≥ 3% 且 未移保本:
    止损价 = 入场价（保证不亏）
    标记已移保本
```

**3. 移动止盈检查**
```
如果 盈利 ≥ 5%:
    记录最高盈利点
    如果 从最高点回撤 3%:
        立即平仓（保住利润）
```

**4. 固定止盈检查**
```
如果 盈利 ≥ 6%:
    立即平仓
    记录盈利
    今日交易次数+1
```

## 第五步：风控体系

### 1. 单笔风控

| 项目 | 设置 | 说明 |
|------|------|------|
| 止损 | 2% | 错单快速止损，小亏离场 |
| 止盈 | 6% | 盈亏比3:1，赚多亏少 |
| 最大亏损 | 账户3% | 单笔风险严格控制 |

**计算示例：**
- 账户1000U，仓位20% = 200U
- 5倍杠杆，2%止损 = 实际亏损200U × 2% × 5 = 20U
- 占账户比例：20U/1000U = 2% ✅

### 2. 日度风控

| 项目 | 设置 | 说明 |
|------|------|------|
| 最大交易次数 | 3次 | 防止过度交易 |
| 最大持仓数 | 2个 | 集中管理，不分散 |
| 日亏损上限 | 15% | 当天停止，保护本金 |

### 3. 余额保护

```
如果 账户余额 < 50USDT:
    停止所有交易
    保留最后本金
    通知用户充值
```

### 4. 自适应调整

| 情况 | 调整 |
|------|------|
| 波动率>3% | 仓位减半 |
| RSI极端(<25或>75) | 仓位减半 |
| 连续亏损2次 | 仓位减半 |
| 连续盈利3次 | 恢复正常仓位 |

## 策略特点总结

### 为什么能赚钱？

1. **高确信信号**：70分门槛过滤掉80%的噪音交易
2. **趋势跟踪**：顺势而为，不逆势操作
3. **小亏大赚**：2%止损 vs 6%止盈，盈亏比3:1
4. **严格风控**：多重保护，活着才有复利

### 为什么不会爆仓？

1. **低杠杆**：5倍杠杆（不是50倍）
2. **紧止损**：2%止损，错单立即砍
3. **仓位控制**：单次20%，不会满仓梭哈
4. **日限次数**：最多3笔，防止上头乱操作

### 为什么适合比赛？

1. **有AI推理**：四维度评分+多条件判断
2. **有明确逻辑**：每一步都有清晰规则
3. **有风控体系**：5层保护，安全优先
4. **可验证**：所有交易带tag，可追溯

## 使用示例

### 场景1：比特币上涨趋势

**市场状态：**
- BTC价格突破前高，成交量放大
- 1小时均线多头排列
- RSI从35涨到55

**AI分析：**
- 趋势分：+40（多头排列）
- 动量分：+15（RSI从超卖反弹）
- 突破分：+30（放量突破）
- 波动率：+5（正常波动）
- **总分：90分** ✅

**AI决策：**
1. 总分90≥70，满足条件
2. 1小时趋势上涨，15分钟也上涨，方向一致
3. 成交量放大1.5倍，确认突破
4. 当前持仓0个，可以开仓
5. 今日交易0次，可以开仓

**执行：**
```bash
# 设置杠杆
okx swap set_leverage --instId BTC-USDT-SWAP --lever 5 --mgnMode cross

# 开仓做多
okx swap place_order --instId BTC-USDT-SWAP --side buy --ordType market --sz 0.01 --tdMode cross --tag agentTradeKit

# 自动设置：
# 止损价 = 入场价 × 0.98
# 止盈价 = 入场价 × 1.06
```

**持仓管理：**
- 价格上涨3% → 止损移到入场价（保本）
- 价格上涨5% → 启动移动止盈
- 价格上涨6% → 触发止盈，平仓获利
- 价格下跌2% → 触发止损，平仓止损

### 场景2：以太坊震荡行情

**市场状态：**
- ETH价格在均线附近纠缠
- 成交量萎缩
- RSI在45-55之间

**AI分析：**
- 趋势分：+20（价格>短期均线）
- 动量分：0（RSI中性）
- 突破分：0（无突破）
- 波动率：-10（波动太小）
- **总分：10分** ❌

**AI决策：**
- 总分10分<70分，不满足开仓条件
- **结论：观望，不操作**

## 错误处理

| 错误 | 原因 | 解决办法 |
|------|------|---------|
| "评分不足" | 市场没趋势 | 等30分钟再分析 |
| "逆势交易" | 大小周期冲突 | 跳过，等方向一致 |
| "已达持仓上限" | 已有2个持仓 | 平仓一个或等待 |
| "今日交易已满" | 已做3笔 | 明天再交易 |
| "余额不足" | 资金<50U | 充值或减少仓位 |
| "API超时" | 网络问题 | 10秒后重试 |

## 安全提示
